# PRD — 사람인 채용 공고 자동 수집기

_작성일: 2026-06-30_

---

## 1. 배경 및 목적

취업 활동 중 매일 사람인에 새로 올라오는 채용 공고를 수동으로 확인하는 것은 번거롭다.
이 스크립트는 매일 점심 무렵 자동으로 공고를 가져와 사용자가 정한 조건에 맞는 것만
로컬 txt 파일에 기록한다. 사용자는 파일을 열어 오늘의 관련 공고를 한눈에 확인할 수 있다.

---

## 2. 사용자 (Target User)

- 로컬 Windows PC에서 Python 스크립트를 직접 실행하거나 Task Scheduler로 자동화하는 개인 사용자
- 특정 직무·지역·경력 조건에 맞는 공고만 모아보고 싶은 구직자

---

## 3. 목표 (Goals)

| # | 목표 |
|---|---|
| G1 | 매일 사람인에서 당일 등록된 신규 공고를 자동으로 가져온다 |
| G2 | 사용자가 `config.py`에 정의한 조건(키워드·지역·경력 등)에 맞는 공고만 필터링한다 |
| G3 | 조건에 맞는 공고를 `output/jobs_all.txt`에 누적 기록한다 |
| G4 | 이미 기록된 공고는 중복 추가하지 않는다 (공고 ID 기준) |
| G5 | Windows 작업 스케줄러로 무인 실행된다 |

---

## 4. 비목표 — Phase 1 제외 기능 (Non-Goals)

다음 기능은 이번 단계에서 구현하지 않는다.

- 이메일·카카오톡·슬랙 등 외부 알림 발송
- 자동 지원 및 이력서 연동
- 사람인 외 다른 채용 사이트(원티드, 잡코리아 등) 연동
- 웹 UI / 대시보드

---

## 5. 사용자 스토리

```
나는 구직자로서,
매일 점심에 PC를 켜지 않아도 자동으로 공고가 수집되어 있어야 하고,
관련 없는 공고 없이 내 조건에 맞는 것만 txt 파일로 쌓여 있기를 원한다.
```

## 6. 기능 요구사항

### 6-1. 공고 조회

- 사람인 Open API (`GET https://oapi.saramin.co.kr/job-search`)를 호출한다.
- `published=1` 파라미터로 당일 등록 공고만 조회한다.
- 한 번 호출 시 최대 110건 조회, 필요 시 페이지네이션으로 전체 수집한다.
- API 호출 실패(4xx/5xx) 시 1회 재시도 후 실패 로그를 남기고 종료한다.

### 6-2. 필터 조건 (`config.py`)

아래 조건을 `config.py`에서 변수로 정의한다. 복수 값은 리스트로 지정한다.

| 변수명 | 설명 | 예시 |
|---|---|---|
| `KEYWORDS` | 공고 제목·직무에 포함된 단어 (OR 조건) | `["Python", "백엔드"]` |
| `LOCATIONS` | 근무 지역 (OR 조건) | `["서울", "판교"]` |
| `CAREER_TYPE` | 경력 구분 | `"경력"` / `"신입"` / `"신입·경력"` / `None` (전체) |
| `EXP_MIN` | 최소 경력 연수 | `1` |
| `EXP_MAX` | 최대 경력 연수 | `5` |

**필터 적용 방식**
- 조건 항목이 `None` 또는 빈 리스트이면 해당 조건을 무시한다 (전체 허용).
- KEYWORDS는 공고 제목 또는 사람인 API `keyword` 필드 중 하나라도 포함하면 통과.
- LOCATIONS는 API 응답의 지역명에 해당 문자열이 포함되면 통과.

### 6-3. 중복 방지

- 수집된 공고 ID를 `output/seen_ids.txt`에 한 줄씩 저장한다.
- 스크립트 실행 시 이 파일을 읽어 이미 수집된 ID는 결과에서 제외한다.
- 신규 공고만 `jobs_all.txt`에 추가하고, 해당 ID를 `seen_ids.txt`에 append한다.

### 6-4. 결과 파일 형식

파일 경로: `output/jobs_all.txt` (없으면 자동 생성, 있으면 append)

공고 1건의 형식:

```
════════════════════════════════════════════════
[수집일] 2024-12-10
[회사]   카카오
[제목]   백엔드 개발자 (Python)
[조건]   서울 강남구 | 정규직 | 경력 1~3년
[직무]   Python, Django, REST API
[링크]   https://www.saramin.co.kr/zf_user/jobs/relay/view?rec_idx=XXXXXXX
[마감]   2024-12-31
════════════════════════════════════════════════
```

- 항목이 없거나 API에서 제공하지 않는 경우 해당 줄을 생략한다.
- 파일 인코딩: UTF-8.

### 6-5. 로그

- 실행 시 콘솔에 수집 건수, 필터 통과 건수, 신규 저장 건수를 출력한다.
- 예: `[2024-12-10 12:05] 조회: 85건 | 필터 통과: 12건 | 신규 저장: 8건`
- 오류 발생 시 오류 내용도 콘솔에 출력한다.

## 7. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| 실행 시간 | API 호출 포함 30초 이내 완료 |
| API 호출 횟수 | 하루 500회 한도 내 유지 |
| 보안 | API 키는 `.env` 파일에만 저장, 코드에 하드코딩 금지 |
| 인코딩 | 모든 파일 입출력 UTF-8 |
| 의존성 | `requests`, `python-dotenv` 외 표준 라이브러리만 사용 |

## 8. 데이터 흐름

```
[사람인 API]
      │
      ▼ (HTTP GET, published=1)
[fetch_jobs.py]
      │
      ├─ 필터링 (config.py 조건 적용)
      │
      ├─ 중복 제거 (seen_ids.txt 비교)
      │
      ├─ output/jobs_all.txt 에 append
      │
      └─ output/seen_ids.txt 에 신규 ID append
```

---

## 9. 성공 기준

- 스크립트를 실행했을 때 조건에 맞는 신규 공고가 `jobs_all.txt`에 올바른 형식으로 추가된다.
- 같은 날 두 번 실행해도 공고가 중복 추가되지 않는다.
- `config.py`의 조건을 바꾸면 다음 실행부터 바뀐 조건이 적용된다.
- Windows 작업 스케줄러에 등록해 무인 실행이 가능하다.
