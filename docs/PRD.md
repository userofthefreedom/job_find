# PRD — 사람인+원티드 채용 공고 자동 수집기

_작성일: 2026-06-30 | 최종 수정: 2026-07-09_

---

## 1. 배경 및 목적

취업 활동 중 매일 사람인·원티드에 새로 올라오는 채용 공고를 수동으로 확인하는 것은 번거롭다.
이 스크립트는 매일 점심 무렵 자동으로 공고를 가져와 사용자가 정한 조건에 맞는 것만
로컬 txt 파일에 기록한다. 사용자는 파일을 열어 오늘의 관련 공고를 한눈에 확인할 수 있다.

> **데이터 소스 선택 배경**: 사람인 공식 Open API는 승인 절차가 까다로워 장기간 대기가
> 필요할 수 있다. 이에 사람인 공개 검색 페이지 스크래핑과 원티드 비공식 API를 조합한다.
> 사람인 공식 API가 승인되면 스크래핑 부분을 공식 API로 교체한다.

---

## 2. 사용자 (Target User)

- 로컬 Windows PC에서 Python 스크립트를 직접 실행하거나 Task Scheduler로 자동화하는 개인 사용자
- 특정 직무·지역·경력 조건에 맞는 공고만 모아보고 싶은 구직자

---

## 3. 목표 (Goals)

| # | 목표 |
|---|---|
| G1 | 매일 사람인·원티드에서 당일 등록된 신규 공고를 자동으로 가져온다 |
| G2 | 사용자가 `config.py`에 정의한 조건(키워드·지역·경력 등)에 맞는 공고만 필터링한다 |
| G3 | 조건에 맞는 공고를 `output/jobs_all.txt`에 누적 기록한다 |
| G4 | 이미 기록된 공고는 중복 추가하지 않는다 (공고 ID 기준) |
| G5 | Windows 작업 스케줄러로 무인 실행된다 |

## 4. 비목표 — Phase 1 제외 기능 (Non-Goals)

다음 기능은 이번 단계에서 구현하지 않는다.

- 이메일·카카오톡·슬랙 등 외부 알림 발송
- 자동 지원 및 이력서 연동
- 사람인·원티드 외 다른 채용 사이트(잡코리아 등) 연동
- 웹 UI / 대시보드

## 6. 기능 요구사항

### 6-1. 공고 조회

두 소스에서 당일 등록 공고를 수집한다.

**사람인 스크래핑**
- 공개 검색 페이지(`https://www.saramin.co.kr/zf_user/search/recruit`)를 `beautifulsoup4`로 파싱한다.
- `days=1` 파라미터로 당일 등록 공고만 필터링한다.
- 페이지네이션으로 전체 수집하며, HTTP 오류 시 1회 재시도 후 해당 소스 건너뜀.

**원티드 비공식 API**
- `https://www.wanted.co.kr/api/v4/jobs` 엔드포인트를 인증 없이 호출한다.
- 최신순 정렬로 최근 공고를 수집하며, offset 기반 페이지네이션으로 전체 수집한다.
- HTTP 오류 시 1회 재시도 후 해당 소스 건너뜀.

> 어느 한 소스가 실패해도 나머지 소스 결과는 정상 처리한다.

### 6-2. 필터 조건 (`config.py`)

아래 조건을 `config.py`에서 변수로 정의한다. 복수 값은 리스트로 지정한다.

| 변수명 | 설명 | 예시 |
|---|---|---|
| `KEYWORDS` | 공고 제목·직무에 포함된 단어 (OR 조건) | `["Python", "백엔드"]` |
| `LOCATIONS` | 근무 지역 (OR 조건) | `["서울", "판교"]` |
| `CAREER_TYPE` | 경력 구분 | `"경력"` / `"신입"` / `"신입·경력"` / `None` (전체) |
| `EXP_MIN` | 최소 경력 연수 | `1` |
| `EXP_MAX` | 최대 경력 연수 | `5` |
| `EXCLUDE_KEYWORDS` | 채용 공고가 아닌 것으로 간주해 제외할 단어 | `["교육생", "무료교육", "설명회", "상시채용"]` |

**필터 적용 방식**
- 조건 항목이 `None` 또는 빈 리스트이면 해당 조건을 무시한다 (전체 허용).
- KEYWORDS는 공고 제목에 포함되면 바로 통과. 제목엔 없고 keyword(직무 태그) 필드에만 포함된 경우,
  제목·고용형태에 EXCLUDE_KEYWORDS가 없을 때만 통과시킨다 — 직무 태그 하나만으로 우연히 걸리는
  무료교육/설명회/포괄적 상시채용 공고 등 실제 채용 공고가 아닌 결과를 걸러내기 위함.
- LOCATIONS는 응답의 지역명에 해당 문자열이 포함되면 통과.

### 6-3. 중복 방지

- 공고 ID에 소스 prefix를 붙여 관리한다: `saramin_XXXXX`, `wanted_XXXXX`.
- 스크립트 실행 시 `jobs_all.txt`를 읽어 이미 수집된 ID는 결과에서 제외한다.
- 신규 공고만 `jobs_all.txt`에 추가한다.

### 6-4. 결과 파일 형식

파일 경로: `output/jobs_all.txt` (없으면 자동 생성, 있으면 append)

공고 1건의 형식:

```
════════════════════════════════════════════════
[ ]
[수집일] 2024-12-10
[출처]   사람인
[회사]   카카오
[제목]   백엔드 개발자 (Python)
[조건]   서울 강남구 | 정규직 | 경력 1~3년
[직무]   Python, Django, REST API
[링크]   https://www.saramin.co.kr/zf_user/jobs/relay/view?rec_idx=XXXXXXX
[마감]   2024-12-31
[ID]     saramin_XXXXXXX
════════════════════════════════════════════════
```

- `[ ]` 줄: 관심 없는 공고를 표시하기 위한 빈 체크 마커. 구분선 바로 다음 줄에 항상 자동 삽입된다.
  사용자가 이 줄을 `[X]`로 바꾸면 다음 실행 시 해당 공고가 파일에서 제거되고 영구 제외 목록에 등록된다.
- `[출처]` 줄: `"사람인"` 또는 `"원티드"` — 항상 출력
- 항목이 없거나 추출할 수 없는 경우 해당 줄을 생략한다.
- 파일 인코딩: UTF-8.

### 6-5. 로그

- 실행 시 콘솔에 수집 건수, 필터 통과 건수, 신규 저장 건수를 출력한다.
- 예: `[2024-12-10 12:05] 조회: 85건 | 필터 통과: 12건 | 신규 저장: 8건`
- 오류 발생 시 오류 내용도 콘솔에 출력한다.

| 항목 | 기준 |

## 9. 성공 기준

- 스크립트를 실행했을 때 조건에 맞는 신규 공고가 `jobs_all.txt`에 올바른 형식으로 추가된다.
- 공고마다 `[출처]` 줄로 어느 플랫폼에서 수집됐는지 확인 가능하다.
- 같은 날 두 번 실행해도 공고가 중복 추가되지 않는다.
- `config.py`의 조건을 바꾸면 다음 실행부터 바뀐 조건이 적용된다.
- Windows 작업 스케줄러에 등록해 무인 실행이 가능하다.
